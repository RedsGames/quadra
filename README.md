# Система расчёта бюджета поездок
**Автор:** [Спиричева Юлия Сергеевна] | **Группа:** [ИС-31] | **Вариант №:** [9]
Консольное приложение для планирования бюджета поездки: учёт расходов по категориям, расчёт итоговой суммы и стоимости на человека, экспорт отчёта в файл.

---

## Возможности

| Функция | Описание |
|---|---|
| Создание поездки | Название, даты, количество участников, стоимость билета |
| Управление расходами | Добавление, редактирование, удаление расходов по категориям |
| Категории расходов | Еда, Одежда, Развлечения, Отель, Такси, Прочее (с кастомным именем) |
| Расчёт бюджета | Транспорт, прочие расходы, общая сумма, стоимость на человека |
| Поиск | Поиск поездки по названию или просмотр всего списка |
| Экспорт | Итоговый отчёт в `.txt`-файл |
| Хранение данных | Персистентный JSON-файл `trips.json` (создаётся автоматически) |
| Ввод дат | Интерактивная маска `ДД.ММ.ГГГГ` с поддержкой Backspace |
| Тесты | `unittest`; проверка валидации и расчётов |

## Использование

### Создание поездки

Выберите пункт `1`, введите данные:

```
Введите название поездки: Сочи 2026
Введите дату начала (ДД.ММ.ГГГГ): 01.06.2026
Введите дату окончания (ДД.ММ.ГГГГ): 10.06.2026
Введите количество человек: 3
Введите стоимость билета туда-обратно на 1 человека: 8500
```

### Добавление расхода

В меню управления поездкой выберите `1 → Добавить новый расход`:

```
Введите сумму расхода: 4500
Выберите категорию расхода:
1. Еда
2. Одежда
3. Развлечения
4. Отель
5. Такси
6. Прочее
Номер категории (1-6): 4
```

### Просмотр итогового отчёта

Выберите `3 → Показать итоговый отчет`:

```
==========================================
           ИТОГОВЫЙ ОТЧЕТ ПОЕЗДКИ
==========================================
Поездка: Сочи 2026
Даты: 01.06.2026 - 10.06.2026
Количество участников: 3
Стоимость билета (на одного): 8500.00 руб.
Всего за билеты (на всех): 25500.00 руб.
------------------------------------------
Прочие расходы по категориям:
  1. Отель: 4500.00 руб.
  2. Еда: 3000.00 руб.
Всего прочих расходов: 7500.00 руб.
------------------------------------------
ОБЩИЕ ЗАТРАТЫ НА ПОЕЗДКУ: 33000.00 руб.
СРЕДНИЙ РАСХОД НА ЧЕЛОВЕКА: 11000.00 руб.
==========================================
```

### Экспорт отчёта

Выберите `4 → Экспортировать итоговый отчет в .txt`. Файл создаётся в корне проекта:

```
Отчет успешно экспортирован в файл: report_сочи_2026.txt
```

### Поиск поездки

Выберите `2 → Найти и редактировать существующую поездку`, введите часть названия или нажмите Enter для показа всех поездок.

### Выход

В любой момент введите `выход` (или `exit`) — программа завершится корректно.

## Архитектура

Проект построен по принципам **Clean Architecture** (слоёная архитектура):

```
 ConsoleForm (CLI)
      │
      ├── CreateTripUseCase    ──► Trip (domain entity)
      ├── UpdateTripUseCase    ──► Trip.update_metadata()
      ├── SearchTripsUseCase   ──► JsonTripRepository.find_by_name()
      ├── AddExpenseUseCase    ──► Trip.add_expense(Expense)
      ├── ManageExpensesUseCase──► Trip.update_expense_amount/category/delete
      ├── GetTripSummaryUseCase──► Trip.calculate_*() → TripSummaryResponse
      └── ExportReportUseCase  ──► TxtReportExporter.export()
                                         │
                                   report_<name>.txt
```

### Поток данных

```
 Ввод пользователя
       ↓
 ConsoleForm._create_trip_flow()
       ↓
 TripCreateRequest (DTO)
       ↓
 CreateTripUseCase.execute()
       ↓
 Trip (domain entity) — валидация в __init__
       ↓
 JsonTripRepository.save()  →  trips.json
```

## Зависимости

Проект использует только стандартную библиотеку Python:

| Модуль | Назначение |
|---|---|
| `json` | Сериализация данных в `trips.json` |
| `datetime` | Работа с датами поездок |
| `re` | Санитизация имени файла при экспорте |
| `unittest` | Автоматические тесты |
| `msvcrt` / `tty` / `termios` | Посимвольный ввод для маски даты |

Требуется **Python 3.10+** (используется синтаксис `X | Y` для типов).
