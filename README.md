# Finance Tracker — Личный финансовый трекер

**Автор:** Дмитриева Евгения Валерьевна | **Группа:** ИС-31 | **Вариант №:** 19

Консольный инструмент на Python для учёта личных финансов: добавление доходов и расходов, фильтрация, подсчёт баланса, экспорт отчёта в `.txt` и визуализация расходов по категориям.

---

## Возможности

| Функция | Описание |
|---|---|
| Транзакции | Добавление доходов / расходов с категорией, датой и описанием |
| Баланс | Автоматический подсчёт: доходы − расходы в реальном времени |
| Пресет-категории | Встроенный список: зарплата, продукты, транспорт, развлечения |
| Фильтрация | По году, месяцу или полной дате, а также по категории без учёта регистра |
| Удаление | Удаление записи по уникальному ID |
| JSON-хранилище | Персистентное хранение в `data/*.json`, файл создаётся автоматически |
| SQLite-хранилище | Альтернативный репозиторий через `sqlite3` (`database.py`) |
| Экспорт | Текстовый финансовый отчёт `.txt` с выровненной таблицей и подписями |
| Визуализация | Круговая диаграмма расходов по категориям (`matplotlib`) |
| Тесты | `pytest`; покрытие баланса, фильтрации, экспорта |

## Структура проекта

```
quadra/
├── src/
│   ├── models.py           # Датакласс Transaction (id, type, category, amount, date, description)
│   ├── services.py         # Бизнес-логика: FinanceService (add, delete, balance, filter, export)
│   ├── storage.py          # JSON-хранилище: JsonStorage (load / save)
│   ├── database.py         # SQLite-хранилище: SQLiteRepository
│   ├── cli.py              # Консольный интерфейс: ConsoleUI
│   ├── visualization.py    # Диаграмма расходов: ChartGenerator
│   └── protocols.py        # Протоколы: ITransactionRepository, IVisualizer
├── tests/
│   ├── test_scripts.py     # Тесты запуска и load_config
│   └── test_report.py      # Тесты баланса, фильтрации, экспорта
├── data/                   # JSON-база данных (создаётся автоматически)
├── main.py                 # Точка входа
└── requirements.txt
```

## Установка

```bash
# 1. Клонировать / распаковать проект
cd quadra

# 2. Создать и активировать виртуальное окружение
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# 3. Установить зависимости
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

При первом запуске программа создаёт папку `data/` и запросит имя файла базы данных:

```
Введите имя файла БД (напр. finance.json):
```

Нажмите Enter, чтобы использовать `finance.json` по умолчанию. После этого открывается главное меню:

```
========================================
 ТЕКУЩИЙ БАЛАНС: 0
========================================
1. Добавить транзакцию
2. Показать все транзакции
3. Фильтрация (дата/категория)
4. Удалить транзакцию по ID
5. Экспорт отчета в .txt
6. Выход
```

```
Сумма: 50000
```

### Просмотр транзакций

```
ID   | Дата         | Тип      | Категория       | Сумма      | Описание
-----------------------------------------------------------------------
1    | 2026-05-01   | +        | зарплата        | 50000.0    | Основная работа
```

## Архитектура

```
main.py  (точка входа)
    │
    ├── JsonStorage          ←── json + os
    │       load(filename)   →  list[dict]
    │       save(filename)   →  записывает в data/*.json
    │
    ├── FinanceService       ←── бизнес-логика
    │       add_transaction()
    │       delete_transaction()
    │       get_balance()         → доходы − расходы
    │       filter_transactions() → по дате и категории
    │       export_summary()      → .txt с таблицей
    │
    ├── ConsoleUI            ←── интерактивное меню
    │       _add() / _show() / _filter() / _delete() / _export()
    │
    ├── SQLiteRepository     ←── альтернативный бэкенд (sqlite3)
    │       add_transaction / get_all / get_by_period / delete_by_id
    │
    └── ChartGenerator       ←── matplotlib
            generate_expense_pie_chart() → .png
```

### Поток данных

```
 Ввод пользователя (ConsoleUI)
         │
         ▼
 FinanceService.add_transaction()
         │
         ▼
 Transaction(id, type, category, amount, date, description)
         │
         ▼
 JsonStorage.save()  →  data/finance.json
         │
         └── при запросе баланса / фильтра:
             FinanceService.get_balance() / filter_transactions()
                         │
                         ▼
                  вывод в ConsoleUI
```
