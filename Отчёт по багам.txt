✅ Сделано:
  - класс QRGeneratorApp с виджетами tkinter
  - метод create_qr_image(): генерация QR через qrcode, конвертация в PIL.Image

🐛 Найден баг #1:
  - при вводе длинного URL (более ~20 символов) программа падала с исключением:
    qrcode.exceptions.DataOverflowError: Data too long for version 1
  - причина: в QRCode(version=1, ...) параметр version=1 ограничивает ёмкость
    кода 17 символами в режиме коррекции ERROR_CORRECT_M
  - получить QR для реальных ссылок типа https://github.com/... было невозможно
  - [статус: исправлен — убран фиксированный version=1, оставлен version=None
    чтобы qrcode сам выбирал нужную версию при make(fit=True)]

🐛 Найден баг #6 (критический):
  - при нажатии кнопки «Выбрать TXT файл» программа немедленно падала с:
    NameError: name 'box_size' is not defined
  - причина: в методе batch_generate() переменная box_size передаётся в
    create_qr_image() на строке 183, но нигде не определяется внутри метода
  - в отличие от generate_single_qr(), здесь не было чтения из self.size_spin
  - пакетная генерация была полностью нерабочей
  - [статус: исправлен — добавлено чтение размера:
    box_size = int(self.size_spin.get()) перед циклом]

🐛 Найден баг #7:
  - при обработке большого TXT-файла (500+ строк) окно приложения
    переставало отвечать: нельзя было двигать, закрыть или свернуть
  - причина: цикл for index, line in enumerate(lines) выполнялся в главном
    потоке tkinter, блокируя event loop на время генерации всех QR-кодов
  - [статус: исправлен — генерация вынесена в отдельный поток threading.Thread;
    прогресс отображается через root.after() обновление Label]

🐛 Найден баг #8:
  - если TXT-файл содержал кириллические ссылки или комментарии в кодировке
    cp1251 (типично для файлов, созданных в Блокноте Windows без BOM),
    программа падала с UnicodeDecodeError при чтении
  - причина: open(txt_path, "r", encoding="utf-8") жёстко задаёт UTF-8,
    не совпадающую с реальной кодировкой файла
  - [статус: частично исправлен — добавлен errors="replace" как запасной вариант;
    в README добавлено требование сохранять TXT в UTF-8]
